# ADR-0003：复用成熟标准，自有核心领域

- 状态：Accepted
- 日期：2026-07-10

## 决策

Totemora 不从头实现所有基础设施，也不把核心领域交给一个现成 Agent 产品。

优先复用以下成熟思想或协议：

- 状态机与持久任务队列：Run 生命周期、取消、重试、恢复。
- 事件日志与 OpenTelemetry：调用、派工、成本和失败观测。
- MCP：工具与部落资产接入。
- A2A：需要时连接外部 Agent。
- AG-UI：Web 实时事件和用户审批边界。
- Git worktree、diff、测试和提交：Coding 的隔离与回滚基础。
- 标准模型 API：Provider 适配层，不绑定单一厂商。

Totemora 必须自有以下模型：

- Member、能力画像、人格、信任和版本。
- Task Analyzer、预算派工和人选解释。
- Mission、Run、评价、失败归因和成长提案。
- Skill/Asset 的采用证据和部落经验。

## 初版约束

v0.2 只搭可替换接口和最小本地实现。例如文件存储可以在产品化阶段换成 SQLite/PostgreSQL，轮询可以换成 AG-UI/SSE，内存调度可以换成成熟队列；领域对象和 Trace 语义不随基础设施替换而改变。

## 重构检查点

完整骨架形成后统一检查：领域边界是否稳定、Runtime 是否过大、文件存储是否达到并发上限、协议适配是否值得引入。检查前不为了“先进”提前迁移，也不继续堆叠明显需要替换的临时代码。
